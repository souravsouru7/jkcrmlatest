1. Executive Summary

JK Interiors Sales CRM is a focused sales management system for an interiors business. Its main purpose is to capture enquiries, track leads through a sales pipeline, schedule follow-ups, manage site visits, record scope sheets, create quotations, and show management-level sales metrics.

The project is split into a Next.js frontend, an Express backend, a Capacitor Android mobile app wrapper, a Google Apps Script integration for Google Sheet leads, and a website enquiry form snippet for direct public lead capture.

Core operating rule: no active lead should exist without a next follow-up date. This is the main CRM discipline the product is designed to enforce.

2. Business Goal

The CRM supports the sales side of an interiors company. The current version concentrates on pre-sales and sales conversion, not execution, installation, procurement, or vendor management.

The intended journey is:

Lead Created
-> Assigned to Sales
-> First Contact
-> Qualified
-> Requirement Collected
-> Site Visit
-> Quotation
-> Negotiation
-> Booking Confirmed
-> Won

5. Login and Authentication

The CRM uses one seeded admin account by default:

Email: admin@jkinteriors.com
Password: Jk@12345

Backend login is handled by POST /api/auth/login. The backend validates the user from MongoDB with bcrypt and returns a JWT token valid for 8 hours. The frontend stores the token using Capacitor Preferences on native Android and localStorage in a browser.

The frontend also includes an offline/demo fallback: if backend login fails but the seeded credentials are entered, it stores an offline-mode token and lets the UI open for review.

6. Backend Architecture

The backend entry point is backend/src/server.js. It creates the Express app, enables CORS, parses JSON, registers routes, connects to MongoDB, seeds the admin user, hydrates the in-memory store from Mongo, and starts the server.

Backend Startup Flow

  1. Load environment variables with dotenv.
  2. Create Express server and apply CORS plus JSON parsing.
  3. Register health, public, auth, leads, follow-ups, site-visits, quotations, dashboard, and reminders routes.
  4. Connect to MongoDB using MONGO_URI.
  5. Seed or repair the admin user.
  6. Load persisted leads and follow-ups into the runtime store.
  7. Listen on the configured port, default 5000.

8. Data Model

Lead

Main sales entity. Fields include id, name, phone, email, location, source, project, budget, estimated budget from sheet, sheet date, DOB, quality fields, call notes, stage, priority, owner, next follow-up, last activity, notes, lost reason, created date, and updated date.

Follow-up

Task linked to a lead. Fields include id, leadId, type, due date, status, outcome, notes, createdAt, and completedAt.

Site Visit

Visit linked to a lead. Fields include id, leadId, date, address, status, notes, assignedTo, createdAt, and optional scopeSheet.

Quotation

Commercial quote linked to a lead. Fields include id, leadId, quotation number, amount, status, validTill, discount, tax, terms, and createdAt.

Scope Sheet

Structured site visit requirement document with project information, service requirements, deliverables, living room, dining, kitchen, bedroom, washroom, civil, carpentry, false ceiling, wall covering, flooring, hardware, electrical, and notes fields.

User

Admin user with email, passwordHash, name, role, active flag, createdAt, and updatedAt.

Persistence note: the current code persists users, leads, and follow-ups through MongoDB. Site visits, quotations, and scope sheets are currently stored in the runtime in-memory store, so they should be moved to MongoDB before production use.

9. Frontend Architecture

The frontend is a Next.js App Router project in frontend/. It uses TypeScript types in frontend/lib/types.ts, an async API client in frontend/lib/api.ts, and storage helpers in frontend/lib/storage.ts. CRM routes are grouped under frontend/app/(crm).

Navigation

Main navigation includes Dashboard, Leads, Pipeline, Follow-ups, Site Visits, Quotations, and Reports. Desktop uses a left sidebar. Mobile uses a bottom navigation bar with More access.

Authentication Hook

useAuth loads the token and user, redirects unauthenticated users to login, and clears storage on logout.

Network and Notifications

The app tracks online/offline state and shows an Offline badge. Notification helpers register push notifications and schedule local notifications on native devices.

11. Sales Workflow Details

  1. Lead Capture: Leads can come from manual entry, website form, Google Sheet, Instagram, Facebook Ads, WhatsApp, phone call, walk-in, referral, marketplace, or other source.
  2. First Follow-up: Manual and public leads automatically create a first call follow-up. Google Sheet leads also create a first call follow-up due the next day.
  3. Qualification: Sales team reviews location, project type, budget, seriousness, timeline, and priority.
  4. Pipeline Progress: Lead moves through New Lead, Contacted, Qualified, Site Visit, Quotation, Negotiation, Won, or Lost.
  5. Site Visit: A scheduled visit moves the lead to Site Visit. Completing a visit updates the lead activity.
  6. Scope Sheet: During or after site visit, detailed measurements, services, materials, and room-wise details are captured.
  7. Quotation: Creating a quotation moves the lead to Quotation. Accepting quotation moves the lead to Won.
  8. Lost Lead: Moving to Lost can store a lost reason for reporting and future analysis.

13. Website Lead Capture

website-form-snippet.html provides a direct enquiry form for any website. Required fields are name and phone. Optional fields include email, location, project type, approximate budget, and message.

The form posts to:

POST {BACKEND_URL}/api/public/lead
Header: x-api-key: {PUBLIC_API_KEY}

The backend protects this endpoint with an API key and a simple in-memory IP rate limit of 10 submissions per hour.

14. Android Mobile App

The frontend/ app is configured for Android through Capacitor with app id com.jkinteriors.crm and app name JK Interiors CRM. The web output directory is out.

Mobile-focused features include native storage through Capacitor Preferences, online/offline detection, push notification registration, local notifications, status bar styling, splash screen configuration, keyboard resizing, and Android build files under frontend/android.

Android Development Commands

cd frontend
npm install
npm run cap:sync
npx cap open android
npx cap run android

For an Android emulator, the API client uses http://10.0.2.2:5000/api to reach the host computer's backend. For real devices, update the API URL to the computer's LAN IP or deploy the backend and set NEXT_PUBLIC_API_URL.

17. Security Review

  • JWT authentication protects internal CRM routes.
  • Password is hashed with bcrypt in MongoDB.
  • Website form uses x-api-key plus rate limiting.
  • Google Sheet sync uses x-sheet-secret.
  • Default secrets are present in code as development fallbacks and must be changed before production.
  • CORS currently allows all origins. For production, restrict this to trusted frontend and Capacitor origins.
  • The frontend demo/offline login path is useful for UI review but should be reviewed before strict production release.

18. Current Limitations

  • Site visits, quotations, and scope sheets are not yet persisted to MongoDB in the current backend code.
  • Only a single seeded admin user is supported by default.
  • No role-based permission matrix is implemented yet.
  • Validation is basic and should be strengthened for production.
  • No automated test suite is visible in the inspected repo.
  • No production logging, audit trail, or rate limiting exists for authenticated routes yet.
  • Google Sheet duplicate detection uses phone number only.
  • Quotation creation tracks summary fields, not full room-wise quote items yet.

19. Recommended Next Development Phases

  1. Persistence completion: add Mongo models and persistence for site visits, quotations, and scope sheets.
  2. Lead detail page: show timeline, follow-ups, visits, scope sheet, quotations, notes, and activity in one place.
  3. Requirement workflow: convert the scope sheet into a complete sales requirement collection flow.
  4. Quotation module: add room-wise items, materials, quantity, GST, discount approval, PDF download, and versioning.
  5. Reporting: add source conversion, stage conversion, monthly bookings, quote conversion, lost reason, and salesperson performance.
  6. Team features: add multiple users, roles, lead assignment, reassignment, and manager views.
  7. Automation: follow-up reminders, quotation expiry alerts, stale lead alerts, and automatic next-action creation.
  8. Production hardening: strict CORS, secure secrets, request validation, error logging, backups, and tests.

20. Final Product Vision

The final CRM should become the central sales cockpit for JK Interiors. Sales executives should know exactly whom to call today, managers should see the pipeline clearly, quotations should be traceable, and every won deal should be ready for handover into a future project execution module.

Once the sales CRM is stable, the natural next modules are project handover, designer assignment, production milestones, installation tracking, payment milestones, and client portal access.
